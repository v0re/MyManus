# SimilarWeb 分析技能

> 本技能利用 SimilarWeb 的公開資料或 API，對任何網站進行深入的流量和參與度分析。

## 核心能力

- **流量指標分析**：估算網站的總訪問量、獨立訪客數和訪問量趨勢。
- **參與度統計**：分析平均訪問時長、每次訪問頁面數和跳出率。
- **全球與分類排名**：獲取網站在全球、特定國家以及其所屬行業類別中的排名。
- **流量來源分析**：識別網站流量的主要來源，包括直接訪問、推薦連結、搜尋引擎、社群媒體和付費廣告。
- **地理分佈**：分析網站訪客的國家/地區分佈。
- **競爭對手比較**：將目標網站與其主要競爭對手在關鍵指標上進行並排比較。

## 使用場景

- **市場研究**：評估一個市場或行業的整體網路流量規模和趨勢。
- **競爭對手分析**：深入了解競爭對手的數位行銷策略和線上表現。
- **投資決策**：將網站流量作為評估公司成長潛力的指標之一。
- **合作夥伴評估**：在建立合作關係前，評估潛在合作夥伴網站的影響力。
- **SEO/SEM 策略制定**：分析流量來源，優化搜尋引擎和付費廣告策略。

## 工作流程

1.  **確定分析目標**：明確要分析的網站 URL 和需要關注的關鍵指標。
2.  **資料獲取**：
    -   **方法 A (API)**：如果擁有 SimilarWeb API 金鑰，直接呼叫 API 獲取結構化資料。
    -   **方法 B (網頁爬取)**：如果沒有 API，則使用瀏覽器自動化工具（如 Playwright）訪問 SimilarWeb 網站，並從公開頁面提取資料。
3.  **資料整理與分析**：將獲取的資料整理成表格，並計算關鍵比率（如各流量來源的佔比）。
4.  **視覺化**：將地理分佈、流量來源等資料轉換為長條圖或圓餅圖，使其更易於理解。
5.  **報告生成**：將所有分析結果、表格和圖表匯總成一份完整的分析報告。

## 範例：分析 `nytimes.com`

**使用者請求**：
> 「請幫我分析一下紐約時報網站 (`nytimes.com`) 的流量狀況。我需要知道它的全球排名、主要流量來源以及訪客的地理分佈。」

**AI 代理執行流程**：

1.  **[INFO]** 收到請求，開始分析 `nytimes.com` 的網站流量。
2.  **[執行中]** 確定使用網頁爬取方式。使用 Playwright 導航至 `https://www.similarweb.com/website/nytimes.com/`。
3.  **[執行中]** 從頁面中提取以下資訊：
    -   全球排名、國家排名、分類排名。
    -   總訪問量、跳出率、每次訪問頁面數、平均訪問時長。
    -   各流量來源（直接、搜尋、社群等）的百分比。
    -   排名前五的訪客國家及其百分比。
4.  **[執行中]** 將提取的資料整理到 Markdown 表格中。
5.  **[執行中]** 使用 `matplotlib` 根據流量來源和地理分佈資料生成圓餅圖，並儲存為圖片檔案。
6.  **[RESULT]** `nytimes.com` 的流量分析報告已完成。報告包含了排名、參與度指標、流量來源和地理分佈的詳細分析及視覺化圖表。請查看附件。
    -   附件：`nytimes_traffic_report.md`, `traffic_sources_pie_chart.png`, `geo_distribution_bar_chart.png`

## 所需工具

- **Python 套件**：`requests`, `beautifulsoup4`, `playwright`, `pandas`, `matplotlib`
- **外部服務**：SimilarWeb 網站或其官方 API

## 注意事項

-   SimilarWeb 的免費公開資料是基於樣本估算的，可能與網站的實際後台資料存在差異。
-   進行網頁爬取時，應遵守網站的 `robots.txt` 規則，並設定合理的請求間隔，避免對伺服器造成過大負擔。
-   如果頻繁使用，建議考慮購買 SimilarWeb 的官方 API 服務，以獲取更準確、更全面的資料。
