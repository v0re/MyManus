# 技能建立器

> 本技能作為一個完整的指南，協助使用者從概念到實作，建立一個高品質、可重用的 AI 代理技能 (Agent Skill)。

## 核心理念

一個好的技能應該是**模組化、可重用、文件清晰且易於維護**的。本技能將引導您遵循最佳實踐，打造出色的技能。

## 技能的組成

一個標準的技能目錄結構如下：

```
[skill-name]/
├── SKILL.md          # 技能定義檔案（必需）
├── scripts/          # 輔助腳本（可選）
├── templates/        # 模板檔案（可選）
└── README.md         # 詳細說明文件（建議）
```

### `SKILL.md`：技能的核心

這是最重要的檔案，它定義了技能的行為和元資料。一個好的 `SKILL.md` 應包含以下部分：

1.  **元資料 (Front Matter)**：
    ```yaml
    ---
    name: your-skill-name
    description: "對技能的簡短描述，說明其用途。"
    version: 1.0.0
    ---
    ```
2.  **標題與簡介**：清晰地命名技能，並用一句話概括其核心價值。
3.  **核心能力**：使用列表詳細說明該技能能做什麼。
4.  **使用場景**：描述在哪些具體情境下應該使用此技能。
5.  **工作流程**：分步說明使用此技能的典型流程。
6.  **範例**：提供一個或多個具體的使用者請求和 AI 代理的執行流程，展示技能的實際應用。
7.  **所需工具/依賴**：列出技能運行所需的外部工具、Python 套件或 API。
8.  **注意事項**：提供使用此技能時需要注意的細節、限制或常見問題。

## 建立技能的步驟

1.  **概念化 (Conceptualization)**
    -   **問題定義**：這個技能要解決什麼問題？
    -   **功能範疇**：它應該包含哪些具體功能？
    -   **命名**：取一個簡潔且能反映功能的名稱（全小寫，用連字號分隔）。

2.  **結構設計 (Structuring)**
    -   建立技能目錄：`mkdir [skill-name]`
    -   建立 `SKILL.md` 檔案。
    -   根據需要建立 `scripts/`, `templates/` 等子目錄。

3.  **撰寫 `SKILL.md` (Authoring)**
    -   **[ASK]** 讓我們開始設計您的技能。請問這個技能的名稱和主要功能是什麼？
    -   根據使用者的回答，逐步填寫 `SKILL.md` 的各個部分。
    -   **[INFO]** 正在為您草擬 `SKILL.md` 檔案...

4.  **開發輔助工具 (Development)**
    -   如果技能需要執行複雜的邏輯，可以將其封裝在 `scripts/` 目錄下的 Python 或 Shell 腳本中。
    -   `SKILL.md` 應說明如何呼叫這些腳本。

5.  **測試與驗證 (Testing)**
    -   設計幾個測試案例，模擬真實的使用者請求。
    -   執行這些測試，確保技能的行為符合預期。
    -   檢查所有文件和範例是否清晰無誤。

6.  **文件化 (Documentation)**
    -   撰寫一份詳細的 `README.md`，提供比 `SKILL.md` 更豐富的資訊，包括安裝指南、進階用法和常見問題解答。

## 範例：建立一個「網站健康度檢查」技能

**使用者請求**：
> 「我想建立一個技能，用來檢查網站的健康度，包括 SSL 憑證狀態、載入速度和基本的 SEO 檢查。」

**AI 代理執行流程**：

1.  **[INFO]** 好的，讓我們來建立一個名為 `website-health-checker` 的技能。
2.  **[執行中]** 建立目錄結構：`mkdir website-health-checker`。
3.  **[執行中]** 撰寫 `SKILL.md` 草稿，包含核心能力（SSL 檢查、速度測試、SEO 檢查）、使用場景等。
4.  **[ASK]** 對於 SEO 檢查，您希望包含哪些具體項目？例如，檢查 `title` 標籤、`meta description` 和 `h1` 標籤是否存在？
5.  **[執行中]** 根據使用者的回饋，完善 `SKILL.md`。
6.  **[執行中]** 開發輔助腳本 `scripts/check_site.py`，該腳本整合 `requests`, `beautifulsoup4` 等工具來執行實際的檢查邏輯。
7.  **[RESULT]** `website-health-checker` 技能已初步建立完成！目錄包含 `SKILL.md` 和一個輔助腳本。您可以開始測試它，或進一步完善文件。請查看附件。
    -   附件：`website-health-checker.tar.gz`

## 最佳實踐

-   **保持單一職責**：一個技能最好只專注於解決一類問題。
-   **提供清晰的範例**：範例是使用者學習如何使用技能的最佳途徑。
-   **版本控制**：在元資料中標明版本號，便於追蹤和管理。
-   **考慮可配置性**：允許使用者透過參數來自訂技能的行為。
